# Бренд-бук GREEN CARD AGENCY

Версия: 1.0  
Дата: 26 May 2026

---

## 1) Платформа бренда

### Название
**GREEN CARD AGENCY**

Правило написания:
- В официальных носителях: `GREEN CARD AGENCY` (капслок).
- В интерфейсах/контенте допускается: `Green card agency` для лучшей читаемости.

### Позиционирование
Мультиязычный digital-сервис пограничного автострахования для B2C клиентов, которым нужно **быстро, прозрачно и безопасно** оформить страхование для пересечения границ.

Формула:
- **Кто:** частные водители и владельцы ТС, пересекающие границу.
- **Что:** помощь в оформлении пограничного страхования.
- **Как:** цифровой путь с понятными шагами, поддержкой и мульти-язычностью.
- **Результат:** минимизация стресса на границе и снижение риска отказа по документам.

### Роль бренда
- Надёжный навигатор в юридически чувствительном процессе.
- «Переводчик» сложного страхового языка в понятные действия.
- Сервис, который снижает тревожность клиента перед поездкой.

### Ценности
1. **Надёжность** — корректность данных и прозрачность процесса.  
2. **Оперативность** — быстрый путь от заявки до связи с менеджером.  
3. **Понятность** — простые формулировки без бюрократического шума.  
4. **Ответственность** — честная коммуникация о рисках и ограничениях.  
5. **Доступность** — мультиязычный сервис и inclusive UX.

## 2) Визуальная система

### 2.1 Логотип и фирменные знаки

#### Базовая версия
- Текстовый логотип: `GREEN CARD AGENCY`.
- Основной акцент — слово **GREEN**.

#### Версии
- Основная: тёмный текст на светлом фоне.
- Инверсия: светлый текст на тёмном фоне.
- Монохром: для документов и технических носителей.

#### Safe zone
- Минимальный отступ вокруг логотипа: высота буквы `G`.

#### Минимальные размеры
- Digital: не менее 120 px по ширине.
- Print: не менее 28 мм.

#### Запреты
- Нельзя растягивать по одной оси.
- Нельзя менять пропорции интервалов между словами.
- Нельзя применять агрессивные тени/обводки.

### 2.2 Цветовая палитра (Design Tokens)

> Токены синхронизировать с CSS variables/дизайн-системой проекта.

#### Core palette
- `--brand-primary`: `#1f7a3a` (глубокий зелёный, доверие/стабильность)
- `--brand-accent`: `#7bae37` (салатовый акцент, действие/энергия)
- `--brand-support`: `#c8f4ef` (светлая бирюза, информирование/partial-state)
- `--brand-success`: `#c9f8a0` (светлый салатовый, success-state)

#### Neutral palette
- `--text-900`: `#111827`
- `--text-700`: `#374151`
- `--text-500`: `#6b7280`
- `--bg-0`: `#ffffff`
- `--bg-50`: `#f9fafb`
- `--border`: `#e5e7eb`

#### Semantic
- `--state-success-text`: `#1b2a12`
- `--state-partial-text`: `#000000`
- `--state-error`: `#b91c1c`
- `--state-warning`: `#b45309`

### 2.3 Градиенты и тени

#### Градиенты
- Hero accent gradient:
  - `linear-gradient(135deg, #1f7a3a 0%, #7bae37 100%)`
- Soft surface gradient (cards):
  - `radial-gradient(720px 240px at 90% 0%, rgba(123,174,55,.16), rgba(123,174,55,0) 62%)`

#### Тени
- `--shadow-1`: `0 8px 24px rgba(17,24,39,.08)`
- `--shadow-2`: `0 20px 60px rgba(0,0,0,.18)`

Принцип: тени мягкие, не «глянцевые», без визуального шума.

## 3) Типографика

### Шрифтовая политика
- Основной UI-шрифт: **Inter** (или системный fallback stack).
- Для кириллицы/латиницы/мультиязычности использовать единый гротеск без декоративных форм.
- Fallback stack:
  - `Inter, ui-sans-serif, system-ui, -apple-system, Segoe UI, Roboto, Arial, sans-serif`

### Типографическая иерархия
- `H1`: 40/46, 800–900
- `H2`: 30/36, 800
- `H3`: 24/30, 700
- `Body L`: 18/28, 400–500
- `Body`: 16/24, 400–500
- `Caption`: 13/18, 500
- `Button`: 15/20, 700

Правила:
- Контраст текста к фону не ниже WCAG AA.
- Не использовать более 3 уровней акцента в одном экране.

## 4) UI-стиль и компоненты

### Кнопки
- Primary: зелёная, высокая контрастность, явный hover/focus.
- Secondary: ghost/outline для вторичных действий.
- Минимальная высота tap target: 44 px.
- Состояния: default / hover / active / disabled / loading.

### Карточки и блоки
- Скругление 16–24 px (в зависимости от блока).
- Светлый фон, мягкая тень, аккуратная рамка.
- Для информационных статусов использовать semantic фоны.

### Сетка и адаптив
- Desktop container: 1200–1280 px max.
- Grid: 12 колонок desktop, 6 tablet, 4 mobile.
- Гэп: 16/24/32 px по брейкпоинтам.
- Принцип mobile-first.

## 5) Тон коммуникации (Tone of Voice)

### Характер голоса
- Профессиональный, спокойный, уверенный.
- Эмпатичный к стрессу пользователя в приграничном сценарии.
- Без канцеляризмов и «продажного давления».

### Как писать
- Короткие, конкретные фразы.
- Объяснять «что нужно сделать сейчас».
- Предупреждать о рисках заранее (документы, сроки, формат файлов).
- Использовать глаголы действия: «Проверьте», «Загрузите», «Подтвердите».

### Как не писать
- Не обещать невозможное («100% гарантия в любой ситуации»).
- Не перегружать юридическими терминами без пояснений.
- Не использовать запугивающую лексику.

## 6) Фото, иллюстрации и графика

### Визуальный принцип
- Реалистичность и доверие: реальные дорожные/пограничные контексты, документы, нейтральные офисные сцены.
- Минимум «стоковой театральности».

### Контент-пакеты в проекте
1. Hero-пакет (международные поездки, ТС, маршрут).
2. Trust-пакет (команда, поддержка, процесс оформления).
3. FAQ/guide-пакет (схемы шагов, чек-листы, форматы документов).

---

## 7) Принципы доступности и UX

- WCAG 2.1 AA как минимум.
- Видимый focus-state на всех интерактивных элементах.
- Контраст текста и кнопок в норме AA.
- Простая структура форм с понятными ошибками.
- Не блокировать отправку заявки при частичной ошибке вложений: информировать и сохранять лид.
- Мультиязычность должна быть консистентной во всех статусов/модалках.

---

## 8) Практическое применение бренд-бука

### Для дизайнеров
- Проверять каждый макет по чеклисту: контраст, токены, иерархия, расстояния, tone.
- Все новые компоненты должны переиспользовать design tokens.

### Для разработчиков
- Использовать только токены, не «хардкодить» случайные цвета.
- Стандартизировать состояния success/partial/error.
- Не добавлять новые UI-компоненты без семантических состояний и a11y.

### Для контент-команды
- Писать по принципу «коротко + по делу + действие».
- Согласовывать юридические формулировки с ответственным менеджером.
- Для каждого языка держать глоссарий ключевых терминов.

---

## 9) Антипаттерны (что ломает бренд)

1. Несогласованные цвета между страницами.
2. Разные формулировки одного и того же статуса в разных формах.
3. «Агрессивные» баннеры/мигающие CTA.
4. Слишком общий контент без конкретных инструкций.
5. Случайные сток-фото, не связанные с продуктом.
6. Несогласованная локализация (разные термины в разных языках).

## 10) Дальнейшее развитие

1. **Brand governance:** ввести версионирование бренд-бука и quarterly review.
2. **Design system maturity:** вынести токены в единый source of truth (Figma Tokens + CSS vars).
3. **Content governance:** сделать multilingual termbase (страховые термины/юридические формулировки).
4. **Trust layer:** системно развивать доказательства экспертизы (кейсы, SLA, процессные сценарии).
5. **SEO+Brand alignment:** каждый новый SEO-кластер запускать через единые бренд-принципы и UX-паттерны.
